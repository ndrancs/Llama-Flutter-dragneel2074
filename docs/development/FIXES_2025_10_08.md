# Fixes Applied - October 8, 2025

## Issues Fixed

### 1. Compilation Error: Undefined `llama_kv_cache_seq_rm` Function

**Error:**
```
error: use of undeclared identifier 'llama_kv_cache_seq_rm'
```

**Root Cause:**
The llama.cpp API was updated. The function `llama_kv_cache_seq_rm()` has been replaced with the new memory management API:
- `llama_get_memory()` - to get the memory object from context
- `llama_memory_seq_rm()` - to remove sequences from memory

**Solution:**
Updated all occurrences in `jni_wrapper.cpp`:

**Before:**
```cpp
llama_kv_cache_seq_rm(g_ctx, 0, -1, -1);
```

**After:**
```cpp
llama_memory_t mem = llama_get_memory(g_ctx);
if (mem) {
    llama_memory_seq_rm(mem, 0, -1, -1);
}
```

**Locations Fixed:**
1. Line ~417: End of `nativeGenerate()` function - clears memory after generation
2. Line ~437: Inside `nativeStop()` function - clears memory when stopping generation
3. Line ~237: Start of `nativeGenerate()` function - NEW addition to clear memory before starting new generation

### 2. Runtime Error: "Failed to decode prompt"

**Error:**
```
Exception: Failed to decode prompt
java.lang.RuntimeException: Failed to decode prompt
at com.write4me.llama_flutter_android.LlamaFlutterAndroidPlugin.nativeGenerate(Native Method)
```

**Root Cause:**
When starting a new generation (especially after a previous one), the KV cache/memory still contained data from the previous conversation. When trying to decode a new prompt with positions starting from 0, there was a conflict with existing data in memory, causing the decode operation to fail.

**Solution:**
Added memory clearing at the **start** of each generation (in addition to clearing at the end). This ensures we always start with a clean slate:

```cpp
// At the beginning of nativeGenerate(), right after validation
llama_memory_t mem = llama_get_memory(g_ctx);
if (mem) {
    llama_memory_seq_rm(mem, 0, -1, -1);
    LOGI("Cleared memory for new generation");
}
```

## Changes Summary

### File: `android/src/main/cpp/jni_wrapper.cpp`

1. **Function: `nativeGenerate()` - Start**
   - Added memory clearing at function start (after validation)
   - Ensures clean state before each generation

2. **Function: `nativeGenerate()` - End**
   - Updated KV cache API call to new memory API
   - Clears memory after generation completes

3. **Function: `nativeStop()`**
   - Updated KV cache API call to new memory API

## API Changes Reference

### Old API (Deprecated)
```cpp
llama_kv_cache_seq_rm(llama_context* ctx, llama_seq_id seq_id, llama_pos p0, llama_pos p1);
```

### New API
```cpp
llama_memory_t llama_get_memory(const struct llama_context* ctx);
bool llama_memory_seq_rm(llama_memory_t mem, llama_seq_id seq_id, llama_pos p0, llama_pos p1);
```

### Parameters:
- `seq_id`: Sequence ID (0 in our case)
- `p0`: Start position (-1 means from beginning)
- `p1`: End position (-1 means to end)
- Using `seq_rm(mem, 0, -1, -1)` clears all data for sequence 0
